Properties palette basic tests (part 5)

Background Controls

background-color on a DIV

  1. New HTML File - desktop hi-fi page - using default Dojo theme (i.e., claro).
  2. Open Visual Editor (any page) and add a DIV. Select the widget. Resize the DIV to a large size (e.g, 300x300px).
  3. Expand the Background Section
  4. Under background-color, type "yellow" into the type-in area. Verify that DIV now has yellow background
  5. Under background-color, select "green" from the dropdown. Verify that DIV now has green background
  6. Under background-color, select "Remove value" from the dropdown. Verify that DIV now has no background
  7. Under background-color, choose "color picker" from the dropdown to bring up the color picker. Pick a color and click OK. Verify that DIV now has the new background color
  8. Under background-color, click on color switch to bring up color picker. Pick a color and click OK. Verify that DIV now has the new background color
  9. Under background-color, select the blank (empty) entry from the dropdown. Verify that DIV now has no background

background-color on a Button using claro

  1. (continuing with previous instructions, which means continue to use claro theme)
  2. Add a dijit.form.Button widget. It should be styled with claro. Select the widget.
  3. Expand the Background Section
  4. Each of the following steps should generate an error prompt saying you cannot modify a read-only theme, and after clicking OK, the background-color value and swatch should be reset to its previous value.
  5. Under background-color, type "yellow" into the type-in area.
  6. Under background-color, select "green" from the dropdown.
  7. Under background-color, click on color switch to bring up color picker. Pick a color and click OK.
  8. Under background-color, click on the "..." button to bring up the Background dialog. Change the background-color on this dialog and click OK.

background-color on a Button using a custom theme

  1. Invoke the New Theme... command to clone claro into a new custom theme (e.g., "myclaro"). OK to leave the new theme as an unchanged copy of claro.
  2. Back in the page editor from previous steps, click on the Switch theme icon and change the desktop theme to "myclaro".
  3. Select the Button widget.
  4. Each of the following steps should generate an "are you sure" prompt warning about making global CSS changes that might affect multiple widgets. Click OK for each prompt. After clicking OK, the color value and swatch should retain their new value and not reset to previous value.
  5. Under background-color, type "yellow" into the type-in area.
  6. Under background-color, select "green" from the dropdown.
  7. Under background-color, click on color switch to bring up color picker. Pick a color and click OK.
  8. Under background-color, click on the "..." button to bring up the Background dialog. Change the background-color on this dialog and click OK.

background-image on a DIV

  1. Select the DIV.
  2. Under background-image, type "url(samples/SampleBanner.jpg). " Verify that DIV now has an image background.
  3. Under background-image, select "" from the dropdown (empty string, first entry). Verify that DIV now has no background image.
  4. Next to background-image, click on the "..." button to bring up the Background dialog. In that dialog, select "image" as the "background-image type", then click on the "..." icon to bring up the file picker and choose "SampleBanner.jpg". Click OK to get out of the dialog(s). Verify that DIV now has an image background.
  5. Try out each of the other background-properties on the Properties palette to verify that it is possible to choose various dropdown menu options and to type values into the text field. Verify that the screen updates to reflect the new values.
  6. Under background-image, click on the "..." button to bring up the Background dialog. In that dialog, verify that the type of background-image is "image", that the URL for the image is showing in the dialog, and that the various detailed options (background-position, etc) have been copied over from the Properties palette. Click OK to get out of the dialog(s). Verify that the Properties palette shows the same values you entered into the dialog and that the screen properly reflects the new property values.

Multiple option changes using Background dialog on a read/write and read/only themes

  1. Select the Button.
  2. Under background-image, click on the "..." button to bring up the Background dialog. In that dialog, click on the "..." icon to bring up the file picker and choose "SampleBanner.jpg". Change the value a couple of other properties (e.g., background-position and background-repeat). Click OK to get out of the dialog. You should see a single dialog warning you about changing a global CSS file. Click OK to proceed. The Button theming should be updated to reflect your changes.
  3. Click on the Switch theme dialog and switch back to "claro".
  4. Under background-image, click on the "..." button to bring up the Background dialog. In that dialog, make sure background-image type is "image". Click on the "..." icon to bring up the file picker and choose "SampleBanner.jpg". Change the value a couple of other properties (e.g., background-position and background-repeat). Click OK to get out of the dialog(s). You should see a single error dialog telling you you cannot change properties on a read-only theme. Click OK to proceed. Verify that the background properties in the properties palette and the screen revert back to claro styling.

Linear Gradients

  1. Select the DIV
  2. Under background-image, click on the "..." button to bring up the Background dialog. In that dialog, select a background-image type of "linear gradient". You should see two color stops, white 0% and black 100%.
  3. For the first color stop, use the menu to choose "blue".
  4. For the last color stop, use the menu to choose "red".
  5. Click on the "+" icon to the right of the first color stop. You should see a new intermediate color stop that is a clone of the first color stop
  6. For the middle color stop, type "green" into the color field and choose "30%" for the position. The color swatch on the top-right of the dialog should now show a red-green-blue gradient.
  7. Click on the "+" icon to the right of the green color stop to clone that stop so there are now 4 color stops. For the 3rd (new) color stop, choose "Color picker" from the menu and pick a yellowish color in the color picker. Set the position value to 60%. The color swatch on the top-right of the dialog should now show a red-green-yellow-blue gradient.
  8. Click on the "-" icon to the right of the green color stop to remove that stop so there are now 3 color stops. The color swatch on the top-right of the dialog should now show a red-yellow-blue gradient.
  9. Set the angle property at the bottom of the dialog to the value: "to right"
  10. Click OK to get out of the dialog(s). The DIV on the screen should have a red-yellow-blue left-to-right gradient.

Radial Gradients

  1. Select the DIV
  2. Under background-image, click on the "..." button to bring up the Background dialog. Due to the previous step, the dialog should say background-image type of "linear-gradient". In that dialog, select a background-image type of "radial gradient". In the color swatch at the top right, you should see a radial gradient with a red center, then yellow then blue as it expands outward.
  3. Set properties as follows: Position:60% 40%, Shape: ellipse, Extent: farthest corner. The color swatch at the top right should update with each of the changes.
  4. Click OK to get out of the dialog(s). The DIV on the screen should have a red-yellow-blue radial gradient.
  5. Save, close, reopen, preview in browser. The radial gradient should appear in both the page editor and in preview.
