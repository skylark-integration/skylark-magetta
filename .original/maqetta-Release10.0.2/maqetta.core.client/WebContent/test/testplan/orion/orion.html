Orion / Maqetta Integration Testing

Create Project from Orion

Steps

  1. Login to Maqetta. Select Open > Orion Navigator to open Orion in a new tab.
  2. In Orion, select New > Folder. Click to open the newly created folder.
  3. Create another new folder here.
  4. Select the new folder by clicking on the same line, but to the right of the folder text (the whole row should be highlighted). Select Actions > Open in Maqetta.
  5. When Maqetta opens, verify that the default project files are listed in the Files view. Also verify that the proper project is open in the project drop down.
  6. Open Sample1.html. Add a widget. Save/preview/close.
  7. Select Open > Orion Navigator to go back to Orion.
  8. Expand the folders you created. Open Sample1.html and verify that the widget you previously added is in the source.

Create New User as Administrator

NOTE: This test is only valid when not using a central authentication server (i.e. LDAP).

Steps

  1. Log out of Maqetta.
  2. Load the URL http(s)://{maqetta_base}:{port}/profile/user-list.html (include /{context} if testing WAR build).
  3. Login as user "admin". You should see a list of registered users.
  4. Click on Create User and create a dummy user.
    • For login, Orion requires a unique, alphanumeric value (no spaces, not email). Make sure to enter the user's email as well. Click Create.
    • If the orion.auth.user.creation.force.email property in maqetta.conf is set to true, then you will have to wait for a confirmation email on that user's email address and click the link to confirm.
  5. Try to login as that user.
    • NOTE: You may want to use a different browser for the dummy user, so you don't have to log out of "admin" account.
  6. Log back in as "admin" and change dummy user's password. Try to login as user again, using new password.
  7. Log back in as "admin" and delete user. Make sure that you cannot log in as dummy user.
