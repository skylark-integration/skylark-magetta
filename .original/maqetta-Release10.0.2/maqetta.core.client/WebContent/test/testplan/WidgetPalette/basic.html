Widget palette basic tests

Overview

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. For each of the composition types (mobile app, desktop app, sketch hifi, sketch lofi), create a new HTML file.
  3. Once the new file is created, the Widget palette is populated.
  4. At the top of the Widget palette is a text input field for searching the widget library.
  5. Widgets are organized into sections and subsections. The sections and subsections differ on the composition type. For mobile apps, you should see:
    • Views
    • Heading
    • Lists
    • Controls (with subsections Buttons; Checkboxes, RadioButtons, Switch; IconContainer; Pickers; Select, Sliders, Spinners; TextBox; Toolbars, ButtonBars; Other)
    • Containers (with subsections IBM One UI [internal only], Dojo and HTML)
    • HTML (with subsections Common; Forms; Headers; Images, Media, Iframes; Paragraphs, Breaks, Rules; Tables)
    • Clip Art (with subsections Arrows; Community; Device Silhouettes; Media; User Interface Controls; User Interface Icons; Other)
    • Drawing Tools
    • Other
    For desktop apps, you should see:
    • Containers (same subsections as mobile)
    • Controls (same subsections as mobile, except no IconContainer or Pickers)
    • HTML (with subsections Common; Forms; Headers; Images, Media, Iframes; Paragraphs, Breaks, Rules; Tables)
    • Clip Art (same subsections as mobile)
    • Drawing Tools
    • Other
    For both sketch types, you should see:
    • Drawing Tools
    • Clip Art (same subsections as mobile)
    • Controls (same subsections as mobile)
    • Containers (same subsections as mobile)
    • HTML (with subsections Common; Forms; Headers; Images, Media, Iframes; Paragraphs, Breaks, Rules; Tables)
    • Other
  6. Click on each widget folder (and subfolder) and observe that the folder opens to display a list of widgets. Make sure that any other open folders (or subfolders) close when you open a different folder (or subfolder).
  7. Go to Controls/Buttons and click on the Button widget in the widget palette to select it. You should see a gray selection rectangle for the Button, with a top right caret icon and a bottom left question mark icon.
  8. Click on the caret icon and you should see a popup that shows alternate button widgets (e.g., Dijit and HTML). Click again on caret icon to close the popup. Click on the caret icon again and then click on the small "x" icon at top right corner of popup to close the popup.
  9. Click on the ? icon to open the help text. The popup should show both a Summary and Description for the Dojo Button widget. Click again on the ? icon to close the help popup. Click on the ? again to re-open the help popup. Click on the small "x" icon at top right of the help popup to close the poup.
  10. Click once again on the caret icon for the Button widget. Click on the "HTML" button" in that popup. The popup should close. Drag the Button widget from widget palette to the canvas. You should see an HTML <button> widget on the canvas. Click once again on the caret icon for the Button widget. Click on the "Dijit" button" in that popup. The popup should close. Drag the Button widget from widget palette to the canvas. You should see an Dojo/Dijit Button widget on the canvas.
  11. With a widget folder open, observe that each widget is highlighted as you hover over it with the mouse.
  12. Select a widget and drag it onto the editor canvas.
  13. Click on a widget to select it, and then click on the canvas. This should also add the widget to the canvas.
  14. Click on the "X" close button to close the html file (without saving).
  15. Observe that the Widget palette is now empty.

Expected results

  1. When no files are open in the Page editor, the Widget palette is empty.
  2. When a file is open in the Page editor, the Widget palette is populated.

Controls

Steps

  1. Choose Create Desktop Application from Create menu
  2. Open the Controls folder (and each of the subfolders) in the Widget palette.
  3. Drag each widget listed in this section onto the canvas. Some of the widgets will allow you to enter data for the widget via a popup text entry control.
  4. If smart input is displayed for the widget enter text that includes HTML for example test <b>smart</b>input
  5. Select the plan text radio button
  6. Click ok button
  7. Validate that the displayed content is correct, in this case test <b>smart</b>input
  8. Preview the file by clicking on the preview icon, validate that the displayed content is correct. Close the preview
  9. Click on the "X" close button to close the html file
  10. Reopen the file, validate that the content is correct in the designer, in this example test <b>smart</b>input
  11. Double click on the widget to display smart input, click the HTML radio button then click ok button
  12. Validate that the displayed content is correct in this example test smartinput
  13. Preview the file by clicking the Preview icon
  14. Verify the displayed content is correct, close the preview.
  15. Click on the "X" close button to close the html file
  16. Reopen the file, validate that the content is correct in the designer, in this example test smartinput

Expected results

  1. You should be able to drag each widget from the Widget palette onto the editor canvas.
  2. As the widget is being dragged, the mouse pointer turns into a + sign and the widget's name.
  3. When the widget is dropped onto the canvas, the widget is displayed on the canvas.

Using the widget filter

Steps

  1. Choose Create Desktop Application from Create menu
  2. In the Widget palette, click on the "Filter..." input field.
  3. Type in some text to search for a widget (e.g. "button").
  4. Select one of the widgets displayed and drag it into the canvas.
  5. Click on the small "x" icon on the right side of the input field to clear the filter field and display the complete list of widgets.
  6. Click on the "X" close button to close the html file (without saving).

Expected results

  1. The small "x" in the filter field should only appear when the filter text field has at least one characters.
  2. When you enter text in the Filter input field, any widgets that contain the text will be displayed. All other widgets will be filtered out.

Show suggested vs show all

Steps

  1. Open at least one file of each composition type: mobile app, desktop app, sketch hi-fi, sketch lo-fi
  2. Open up the menu at the top/right of the widget palette. You should see two menu items, "Suggested widgets" and "All widgets", and by default "Suggested widgets" should be checked.
  3. For each of the four composition types, in the Widget palette, open up Controls/Buttons section. Click on "Button" to select it. Click on the caret icon on top/right to show the list of alternative widgets.
    • For mobile app, you should see Dojo Mobile and HTML, but not Dijit.
    • For desktop app, you should see Dijit and HTML, but not Dojo Mobile.
    • For the two sketch modes, you should see Dijit, Dojo Mobile and HTML.
  4. Choose the "All widgets" option from the menu. After selecting it, look at the menu again. The "All widgets" option should be checked now.
  5. For each of the four composition types, in the Widget palette, open up Controls/Buttons section. Click on "Button" to select it. Click on the caret icon on top/right to show the list of alternative widgets.
    • For mobile app, you should see Dojo Mobile, Dijit and HTML in that order.
    • For desktop app, you should see Dijit, Dojo Mobile and HTML in that order.
    • For the two sketch modes, you should see Dijit, Dojo Mobile and HTML.

Expected results

  1. As described above.
