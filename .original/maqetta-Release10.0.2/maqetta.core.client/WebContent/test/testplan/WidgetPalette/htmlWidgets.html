Widget Palette: HTML Element Tests

Media (<audio>, <video>, and <embed>)

Steps

  1. Choose New HTML File from the New menu
  2. Open the HTML folder in the Widget palette.
  3. Drag an <audio> widget onto the canvas.
  4. Notice a Smart Input dialog is displayed with a text field. By default, it will contain the following: 
    http://maqetta.org/samples/media/maqettaSound.mp3
http://maqetta.org/samples/media/maqettaSound.ogg
  5. Click OK.
  6. Click on the Preview icon in the toolbar. This should open a new browser window showing your <audio> element. You should hear sound when you click play.
  7. Double-click the <audio> element to bring Smart Input back up.
  8. Enter the file name for a non-existent file (e.g. this_is_a_bad_filename) and click OK. Ensure the <audio> element still has a size and is available for selection. (In Chrome, there will be partial playback controls. And, in FF, there will just be an empty box.)
  9. Double-click the <audio> element to bring Smart Input back up and specify a new source by copying and pasting the following into the text box: 
    http://www.w3schools.com/html/horse.mp3
http://www.w3schools.com/html/horse.ogg
    NOTE: None of the commonly used audio formats are supported by all of the major browsers (see Audio Formats and Browser Support). It is expected that specifying both mp3 and ogg files (like we do above) should work across browsers. But, if you try some of your own audio files, please note possible incompatibilities with your browser.
  10. Click on Preview icon in the toolbar. This should open a new browser window showing your audio element. You should hear sound when you click play.
  11. EXTRA CREDIT:
    1. Drop a <video> element onto the page.
    2. Notice a Smart Input dialog is displayed with a text field. By default, it will contain the following: 
      http://maqetta.org/samples/media/maqettaVideo.mp4
http://maqetta.org/samples/media/maqettaVideo.ogg
    3. Click OK.
    4. Click on the Preview icon in the toolbar. This should open a new browser window showing your <video> element. You should see a video and hear sound when you click play.
    5. Double-click the <video> element to bring Smart Input back up and specify a new source by copying and pasting the following into the text box: 
      http://www.w3schools.com/html/movie.mp4
http://www.w3schools.com/html/movie.ogg
      NOTE: None of the commonly used video formats are supported by all of the major browsers (see Video Formats and Browser Support). It is expected that specifying both mp4 and ogg files (like we do above) should work across browsers. But, if you try some of your own video files, please note possible incompatibilities with your browser.
    6. Click OK.
    7. Click on the Preview icon in the toolbar. This should open a new browser window showing your <video> element. You should see a video and hear sound when you click play.
  12. DOUBLE EXTRA CREDIT:
    1. Drop an <embed> element onto the page.
    2. Notice a Smart Input dialog is displayed with a text field. By default, it will contain the following: 
      http://maqetta.org/samples/embed.html
    3. Click OK.
    4. Double-click the <embed> element to bring Smart Input back up and specify a new source by copying and pasting the following into the text box: 
      http://www.w3schools.com/html/helloworld.swf
    5. Click OK.
    6. Click on the Preview icon in the toolbar. This should open a new browser window showing your <embed> element. Assuming you have Flash installed for your browser, you will see a small Flash animation that says "Hello, World."
  13. Click on the "X" close button to close the html file (without saving).

Expected results

  1. See above.

<select>

Steps

  1. Create new HTML file.
  2. Drag a <select> onto the canvas.
  3. Smart Input should be displayed with 3 items. Click OK, and a drop down button should appear on the canvas.
  4. Go to widget specific properties and check the multiple attribute. The drop down should become a select list.
  5. Still within widget specific properties, enter 15 into the size field. This will cause the select field to increase in vertical size to accommodate 15 rows of data without scrolling.
  6. Double-click the select list to bring up Smart Input. Copy and paste the text below into the text box and click OK. 
    Group 1
>Item 1
>+Item 2
Group 2
>Item 3
>Item 4
Group 3
>Item 5
>Item 6	
Item 7
Item 8
Item 9
Item 10
  7. As shown in the screen shot below, there should be three option groups (each containing 2 items), the last 4 items should be top-level items (not option groups), and Item 2 should be selected.
  8. Click on the "X" close button to close the html file (without saving).

Expected results

  1. See above.

<fieldset> and <input>

Steps

  1. Choose New HTML File from the New menu
  2. Open the HTML folder in the Widget palette.
  3. Drag a <fieldset> widget onto the canvas.
  4. Notice a Smart Input dialog to enter legend text is displayed. Change the value from "Legend" to one of your choosing (say "My Fieldset Legend") and click OK.
  5. Resize the <fieldset> so it has a width of roughly 400px and a height of roughly 200px.
  6. Drag an <input> onto the canvas inside of the <fieldset>. You should see a text field rendered inside of the <fieldset>.
  7. Under widget-specific properties, put a check next to disabled and change the value field to something like "my disabled input". You should see your text in a disabled text field.
  8. Drag another <input> onto the canvas inside of the <fieldset>. You should see another text field next to the first one.
  9. With the 2nd <input> selected, choose "file" from the type pull-down list under widget specific properties. The <input> that was a text field should turn into a file chooser.
  10. Click on Preview icon in the toolbar. This should open a new browser window showing your fieldset (with your caption) containing two inputs. The first input should be a disabled text field. The second input should be a file chooser (click the "Choose File"" button to confirm it brings up a file dialog).
  11. Click on the "X" close button to close the html file (without saving).

Expected results

  1. See above.
