Projects and Project Templates Testing

New user - create default project

Steps

  1. If possible, run a version of Maqetta where you are able to create a brand new login.
  2. Open the Files palette. Look at the toolbar and show the contents of the projects dropdown menu.
    • The DropDownSelect widget next to the label "Project:" should say "project1"
    • The DropDownSelect widget's menu should have one entry: "project1"
  3. Double-click on app.css, add a style rule to the bottom such as #foo { color:red; }, click on Save, then close the editor for app.css..
  4. From the menu at the top/right corner of the Files palette, issue a "New project..." command.
    • The New Project dialog should appear
    • The three checkboxes should be unchecked by default.
    • The dropdown menu next to "User project template" should be disabled.
    • The Create button should be disabled until at least one character is typed into the name field
  5. Type various alphas, numbers and punctuation characters into the name field.
    • Only alphas, numbers, hyphens and periods should be allowed
      • the name field
    • Whenever an invalid character is entered, a red exclamation point should appear on the right side of type-in field and Create button should be disabled
  6. Enter any valid string for the name field and click Create.
    • Maqetta should refresh the browser page
    • A default project should appear (with app.js, app.css, lib, samples, themes)
    • The project dropdown menu should show the project name that you just typed.
  7. Double-click on app.css to open the source editor.
    • You should see the original version of app.css - the style rule you added above should not be in the file.
  8. Close the editor for app.css.

Clone existing project

Steps

  1. Issue a New Folder command and create a new folder (e.g., folder1).
  2. Double-click on app.css, then SaveAs to file1.css into folder1, then close the editor for file1.css..
  3. Double-click on app.css again, add a style rule to the bottom such as #foo { color:red; }, click on Save, then close the editor for app.css.
  4. From the menu at the top/right corner of the Files palette, issue a "New project..." command.
  5. Turn on the checkbox that says "Copy files from current project into new project".
  6. Enter any valid string for the name field and click Create
    • Maqetta should refresh the browser page
    • The new project should have both the standard files (app.js, app.css, lib, samples, themes), but should also have folder1 with file1.css. inside of folder1.
    • The project dropdown menu's current value should be the project name that you just typed.

Rename and delete projects

Steps

  1. From the menu at the top/right corner of the Files palette, issue a "Rename this project..." command. Enter a new name for the current project.
    • Maqetta should refresh the browser page
    • The new project should have the same contents as previously.
    • The project dropdown menu's current value should show the rename project.
  2. From the menu at the top/right corner of the Files palette, issue a "Delete this project..." command.
    • An "are you sure" confirmation dialog should appear
  3. Click OK on the confirmation dialog.
    • Maqetta should refresh the browser page
    • One of the other projects should become the current project (e.g., project1).
    • The project dropdown menu's current value should show a different project, and the deleted project should not appear on list of projects on the dropdown menu.

Create new project template - unshared

Steps

  1. Issue a New Folder command and create a new folder (e.g., a1).
  2. With a1 selected in Files palette, issue a "Create CSS File" command and create file a1.css in folder a1, then close the editor for a1.css..
  3. Issue a New Folder command and create a new folder (e.g., b1).
  4. With b1 selected in Files palette, issue a "Create CSS File" command and create file b1.css in folder b1, then close the editor for a1.css..
  5. Double-click on app.css, add a style rule to the bottom such as #aaa { color:red; }, click on Save, then close the editor for app.css.
  6. From the menu at the top/right corner of the Files palette, issue a "Save as project template..." command.
    • The "Save as project template" dialog should appear
    • The "Share with everyone" checkboxes should be unchecked by default.
    • The Create button should be disabled until at least one character is typed into the name field
  7. Type various alphas, numbers and punctuation characters into the name field.
    • Only alphas, numbers, hyphens and periods should be allowed
      • the name field
    • Whenever an invalid character is entered, a red exclamation point should appear on the right side of type-in field and Create button should be disabled
  8. Enter any valid string for the name field (e.g., template1) and click Create.
    • An alert should appear confirming that the project template was created.
  9. In the Files palette, delete folder b1.
  10. From the menu at the top/right corner of the Files palette, issue a "New project..." command.
  11. Enter a name for the new project (e.g., projectX), click the checkbox for "Use project template", choose the project template you just created from the menu (e.g., template1), and click Create. Because you used a project template but did not clone the previous project:
    • Maqetta should refresh the browser page
    • The current project should be "projectX" (or whatever you typed)
    • Folder b1 should be there. (Copied from the template).
    • The new style rule that you entered into app.css should be there (because it was in the template).
  12. In a different browser, login to the same Maqetta server using a different ID. Issue a "New project..." command.
    • The project template you just created should not appear in the list of project templates (because that project template was not "shared with everyone").
  13. Go back to the browser with your original Maqetta session.

Create new project template - use template and also clone

Steps

  1. (Assumes folders a1 and b1 are present due to previous steps in test)
  2. Delete folder a1.
  3. Double-click on b1/b1.css and type some different content into that file.
  4. Issue a New Folder command and create a new folder (e.g., c1).
  5. With c1 selected in Files palette, issue a "Create CSS File" command and create file c1.css in folder c1, then close the editor for c1.css..
  6. From the menu at the top/right corner of the Files palette, issue a "New project..." command.
  7. Enter a name for the new project (e.g., projectY), click the checkbox for "Use project template", choose the project template you just created from the menu (e.g., template1), also click the checkbox that says "Copy files from current project into new project", and click Create. Because you both used a project template and cloned the current project:
    • Maqetta should refresh the browser page
    • The current project should be "projectY" (or whatever you typed)
    • Folder a1 should be there. (Copied from the template).
    • Folder b1 should be there and file b1.css should show the changes you made earlier in this section (because current project contents overwrite same-named files from the template).
    • Folder c1 should be there. (Cloned from previous project).

Create new project template - shared

Steps

  1. (Assumes folders a1 and a1 are present due to previous steps in test)
  2. From the menu at the top/right corner of the Files palette, issue a "Save as project template..." command. Enter a template name (e.g., template2), turn on the "Share with everyone" checkbox, and click Create.
    • An alert should appear confirming that the project template was created.
  3. In a different browser, login to the same Maqetta server. (If already logged in, then do a browser refresh.) Issue a "New project..." command.
    • The project template you just created should not appear in the list of project templates (because that project template was "shared with everyone").
  4. Go back to the browser with your original Maqetta session

Manage project templates

Steps

  1. (Assumes two project templates have been created by previous steps (e.g., template1 and template2).
  2. From the menu at the top/right corner of the Files palette, issue a "Manage project templates..." command.
    • The Manage project templates dialog should appear, with a tabular display of your project templates.
  3. For one of the project templates, rename it and change its sharing value. To rename, click on the name field. An inline edit control should appear. (You may have to hit Enter to make the change "take".) Click on the sharing checkbox to change its value.
    • Changed values should have a pink background.
  4. For another of the templates, click on the DEL button 3 times.
    • With the first click, the row should get a pink background and all text should have strike-though.
    • With the second click, the highlighting should go away (because you have "un-deleted" that template).
    • With the third click, the row should get a pink background and all text should have strike-though.
  5. Click on the Update button to commit your changes.
    • A confirmation dialog should appear that summarizes the changes you are about to make..
  6. Click OK on the confirmation dialog.
  7. Bring up the Manage project templates dialog again.
    • The dialog should show the template name change, the sharing change, and the deleted project should no longer appear.
