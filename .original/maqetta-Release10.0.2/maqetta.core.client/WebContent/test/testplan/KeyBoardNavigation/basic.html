Keyboard Navigation basic tests

{Accl} is the default platform keyboard accelerator
  • Windows/Linux: Control
  • Mac OS X: Command

Global Keyboard Navigation

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary.
  2. Open the New Desktop Application dialog
  3. Press Enter and a new file should be created.
  4. Click the left mousebutton on the empty canvas of the new file.
  5. Press {Accl} o (letter O) and the Open File dialog will show up.
  6. Press ESC and the dialog should go away.

Expected results

  1. These steps should work no matter where the focus is

Visual Editor Keyboard Navigation

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary.
  2. Create a new desktop application file
  3. Drag and drop a Dijit Button
  4. In the smart input, change the label of the button and press OK.
  5. Click on the canvas of the Visual Editor
  6. {Accel} z should undo and {Accel} Shift Z should redo.
  7. Click on the Maqetta banner at the top of the page. The above keyboard commands should not do anything.
  8. Now click on the button and press the DELETE key - that should delete the button.
  9. Click again on the canvas and press {Accel} 1-4: this should toggle between all 4 display modes.
  10. Click on the Maqetta banner and press {Accel} 1-4: this should toggle between all 4 display modes.

Expected results

  1. Undo/Redo should only work when the canvas of the Visual Editor has focus.
  2. Display mode keys should work anywhere in the application

Files Palette Keyboard Navigation

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary.
  2. Click on any item in the Files Paletta and press the DELETE key - you should get a dialog confirming you want to delete the file.
