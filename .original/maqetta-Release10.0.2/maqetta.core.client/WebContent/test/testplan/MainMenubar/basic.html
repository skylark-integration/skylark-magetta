Main menubar basic tests

Note: instead of running this test suite manually you can use the automated seleniumIDE test suite - test/seleniumIdeTestCases/mainMenuOverview/main menu test suite

Create menu: create desktop application - default filename, default options, no save

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. From the Create menu on the banner, choose "Desktop Application..."
  3. Click Create to accept the default file name (file1.html)
  4. Drag/drop any widget onto canvas
  5. Close the editor by clicking "x" on editor's tab

Expected results

  1. After clicking OK on file dialog, editor window appears with a blank canvas, and...
  2. Files palette should show the file name for this file, and...
  3. Widget palette is populated with many widgets organized into categories (e.g., Form, Layout, etc), and...
  4. Outline palette contains a single entry (e.g., ./file1.html), and...
  5. Property palette appear but entries are disabled until a widget is selected, and...
  6. After closing, the file name should be removed from the Files palette.

Create menu: create desktop application - default filename, with save

Steps

  1. From the Create menu on the banner, choose "Desktop Application..."
  2. Click Create to accept the default file name (file1.html)
  3. Drag/drop any widget onto canvas
  4. Click the "Save" button on editor toolbar
  5. Close the editor by clicking "x" on editor's tab
  6. Double-click on the filename in Files palette to reopen
  7. Close the editor by clicking "x" on editor's tab

Expected results

  1. After clicking Create on file dialog, ...(same as above)...
  2. After clicking save, dirty indicator (*) should disappear from editor's tab.
  3. (With 0.6, there is a known bug with dirty logic where you are prompted "are you sure" even though no changes since last save)
  4. After closing, the file name should continue to be visible in Files palette.
  5. After reopening, the visual page editor should show the widget you added to the page.

Create menu: create desktop application - nested directory, with save

Steps

  1. From the Create menu on the banner, choose "Folder..."
  2. Type a name for the new folder or accept the default name. Keep the current parent folder as ".".
  3. Highlight the new folder name in the Files palette.
  4. From the Create menu on the banner, choose "Desktop Application...:
  5. You should see your folder name listed as "parent folder". Enter a filename and click OK.
  6. Drag a button widget onto the canvas.
  7. Click the "Save" button on editor toolbar
  8. Close the editor by clicking "x" on editor's tab
  9. Double-click on the filename in Files palette to reopen
  10. Close the editor by clicking "x" on editor's tab

Expected results

  1. A new editor window should appear (as with previous test sections).
  2. In the Files palette, the new file should appear as a nested file within the folder you just created..
  3. The button should be properly styled using the default theme (which in most cases is Claro, which results in bluish styling)

Create menu: Create CSS File - different filename in root directory, with save

Steps

  1. From the Create menu on the banner, choose "CSS File..."
  2. Don't accept the default filename. Instead, type a new file name, but within the default directory.
  3. Add some text to the file, such as ".foo { color:red; }"
  4. Click the "Save" button on editor toolbar
  5. Close the editor by clicking "x" on editor's tab
  6. Double-click on the filename in Files palette to reopen
  7. Close the editor by clicking "x" on editor's tab

Expected results

  1. The new CSS file should be visible in the Files palette and upon reopening show the content you entered.

Create menu: Create theme

Steps

  1. From the Create menu on the banner, choose "Theme..."
  2. Select "claro" and enter a new theme name, such as "mytheme1", and click OK
  3. Look under themes/ folder in Files palette and look for a folder with the name you have provided.

Expected results

  1. The theme editor should open.
  2. There should be a new folder themes/mytheme1/, inside of which should be many CSS files and files mytheme1.json and mytheme1.theme.

Open menu: Theme Editor...

Steps

  1. From the Open menu on the banner, choose "Theme Editor..."
  2. In dialog that appears, choose the theme you created earlier (e.g., mytheme1).

Expected results

  1. After issuing the Open Theme Editor command, the theme editor should have appeared, showing a collection of Dojo widgets, and...
  2. Properties palette should have content, and...
  3. States palette should show a list of widget states, such as Normal, Selected, and Hover, and...
  4. Widgets and Outline palettes should be empty

Settings Menu: Preferences

Steps

  1. From the Settings menu on the banner, choose Preferences...

Expected results

  1. As of now, there should be two menu items on the Settings menu, Preferences... and Theme sets...
  2. Preferences dialog should appear. There should be a tree widget on the left with two major sections, Project and HTML. Inside of the HTML, there should be a single entry, Visual Editor. The Visual Editor section should show various options, including Snap to nearest widget (default:on) / When dragging, show possible parents (default:off) / Warn on CSS Override (default:on) / z-index for sketch mode (default:900).
  3. (Note that there is a separate test plan for Theme sets...)

Help Menu

Steps

  1. From the Help menu on the banner, select each of the menu items (Documentation, Tutorials, About Maqetta).

Expected results

  1. Documentation and Tutorials menu items should open a new browser tab showing the main Maqetta docs page and the tutorials subsection, respectively.
  2. About Maqetta should open a dialog displaying the version and build information.
