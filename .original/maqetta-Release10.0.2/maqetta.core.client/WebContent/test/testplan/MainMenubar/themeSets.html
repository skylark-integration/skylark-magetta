Theme sets test plan

Note instead of running this test suite manually you can use the automated seleniumIDE test suite - test/seleniumIdeTestCases/themeSets/ThemeSetsTestSuite.html

Add new theme set.

Steps

  1. Choose New Theme from the main menu.
  2. Select claro from the drop down list Theme to clone.
  3. Enter claro2 in the new name box.
  4. When the theme editor is displayed select the global widget and change the background color. Save the theme and close the theme editor.
  5. Choose New Theme from the New menu.
  6. Select custom from the drop down list Theme to clone.
  7. Enter custom2 in the new name box.
  8. When the theme editor is displayed select the global widgets and change the background color. Save the theme and close the theme editor.
  9. Choose settings->Theme sets from the main menu.
  10. From the Manage theme sets dialog, click on the + below the Theme sets: selection list. A theme set called site_dojo_default_1 Should be added to the list of themes.
  11. With the site_dojo_default theme set still selected in the Theme sets: selection list click on the + a theme set called site_dojo_default_2 should be added to the list.
  12. Click-select on the the site_dojo_default_2 theme set in the Theme sets: selection list, then click on the + a theme set called site_dojo_default_2_1 should be added to the list.
  13. Click Save.
  14. Reopen Manage theme sets Settings -> Theme sets

Expected results

  1. Theme sets: selection list should have the following theme sets.
    1. site_dojo_default
    2. site_dojo_default_1
    3. site_dojo_default_2
    4. site_dojo_default_2_1

Remove theme sets unsaved

Steps

  1. Continuing from the test case above, with the Manage theme sets dialog displayed.
  2. Select site_dojo_default_2 in the Theme sets: selection list.
  3. Click on the "x" below the Theme set: selection list, site_dojo_default_2 should be removed from the list.
  4. Select site_dojo_default_2_1 in the Theme sets: selection list.
  5. Click on the "x" below the Theme set: selection list, site_dojo_default_2_1 should be removed from the list.
  6. Click Cancel button.
  7. Reopen Manage theme sets Settings -> Theme sets

Expected results

  1. Theme sets: selection list should have the following theme sets.
    1. site_dojo_default
    2. site_dojo_default_1
    3. site_dojo_default_2
    4. site_dojo_default_2_1

Rename theme sets

Steps

  1. Continuing from the test case above, with the Manage theme sets dialog displayed.
  2. Select site_dojo_default_2 in the Theme sets: selection list.
  3. Click on the Rename button.

  5. Click Rename

Modify theme sets

  2. Select my_theme_set in the Theme sets: selection list.
  3. Select claro2 in the Dojo desktop 1.7 theme: drop down.
  4. Select custom2 in the Dojo mobile 1.7 theme: drop down.
  5. Select site_dojo_default in the Theme sets: selection list.
  6. Click the "+" below the Theme sets: selection list.
  7. Select site_dojo_default_1 in the Theme sets: selection list.
  8. Select (device-specific) in the Dojo mobile 1.7 theme: drop down.
  9. Select android from the Android drop down list.
  10. Select custom2 from the Blackberry drop down list.
  11. Select ipad from the iPad drop down list.
  12. Select iphone from the iPhone drop down list.

  15. The Rename theme set dialog should be displayed, enter dojo_theme_set in the text box.

  17. Click Save to save the theme sets.
  18. Reopen Manage Theme sets dialog Settings->Theme sets.

Applying theme sets to HTML

Steps

  1. Continuing from the test case above, create a new html file with a iphone composition type.
  2. Drag and drop a mobile Heading widget on the canvas.
  3. Using the gear icon for the page editor toolbar, choose the Switch theme command. (Note: you will probably be asked to save your document before the dialog comes up.)
  4. The theme switching dialog should be displayed with the Theme set: text box displaying (none)
  5. Select site_dojo_default from the Theme sets drop down: The following should be displayed:
    1. claro should be selected but disabled in the Dojo desktop drop down.
    2. custom should be selected but disabled in the Dojo mobile drop down.
    3. custom should be displayed but disabled in the Android drop down.
    4. custom should be displayed but disabled in the Blackberry drop down.
    5. custom should be displayed but disabled in the iPad drop down.
    6. custom should be displayed but disabled in the iPhone drop down.
    7. custom should be displayed but disabled in the Other drop down.
  6. Select my_theme_set from the Theme sets drop down, the following should be displayed:
    1. claro2 should be selected but disabled in the Dojo desktop drop down.
    2. custom2 should be selected but disabled in the Dojo mobile drop down.
    3. custom2 should be displayed but disabled in the Android drop down.
    4. custom2 should be displayed but disabled in the Blackberry drop down.
    5. custom2 should be displayed but disabled in the iPad drop down.
    6. custom2 should be displayed but disabled in the iPhone drop down.
    7. custom2 should be displayed but disabled in the Other drop down.
  7. Select dojo_theme_set from the Theme sets drop down, the following should be displayed:
    1. claro should be selected but disabled in the Dojo desktop drop down.
    2. (device_specific) should be selected but disabled in the Dojo mobile drop down.
    3. android should be displayed but disabled in the Android drop down.
    4. custom2 should be displayed but disabled in the Blackberry drop down.
    5. ipad should be displayed but disabled in the iPad drop down.
    6. iphone should be displayed but disabled in the iPhone drop down.
    7. iphone should be displayed but disabled in the Other drop down.
  8. Click Select button

Expected results

  1. Click on the Display source icon from the designer toolbar, examine the source you should see the following in the data-dojo-config parameter of the dojo.js script link:
    1. 'themeMap':[['Android','',['themes/android/android.css']],['BlackBerry','',['themes/custom/custom.css']],['iPad','',['themes/ipad/ipad.css']],['iPhone','',['themes/iphone/iphone.css']],['.*','',['themes/iphone/iphone.css']]],'mblThemeFiles':[]

    6. iphone should be displayed but disabled in the iPhone drop down.
