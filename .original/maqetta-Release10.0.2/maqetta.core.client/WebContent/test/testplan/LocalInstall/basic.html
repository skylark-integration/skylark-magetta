Local install basic tests

Pull down the most recent Maqetta build

Steps

  1. Download and unpack the latest version of the Maqetta server package.

Expected results

  1. The directory structure should look something like this:
    • artifacts.xml
    • configuration
    • eclipse-orion-notice.html
    • features
    • maqetta.conf
    • maqetta.server.mac.command
    • maqetta.server.sh
    • maqetta.server.win.bat
    • notice.html
    • p2
    • plugins
    • readme.html
    • siteConfig

Launch the Maqetta server (need to run tests on both Windows and Mac)

Steps

  1. With the Explorer (Windows) or Finder (Mac), navigate to the directory into which you unzip-d the Maqetta files
  2. Launch the Maqetta server process:
    • On Windows, double-click on the file maqetta.server.win.bat (Depending on your settings, the Windows explorer might not show the ".bat" extension)
    • On Mac, double-click on the file maqetta.server.mac.command (Depending on your settings, the Mac finder might not show the ".command" extension)
  3. Try to connect to the server. Default URL is http://localhost:50000/maqetta.

Expected results

  1. You can tell that the server has launched if you see the following message in the shell window: 
    !!!!!!!!!!!!!!
NOTE: CLOSING THIS WINDOW WILL 
      STOP THE MAQETTA SERVER PROCESS
!!!!!!!!!!!!!

Using directory: .....

osgi>
  2. The Maqetta app should appear in the browser when loading the URL.
  3. The Maqetta server will continue to run until you close the shell window that comes up after you double-click on the server launch file. Therefore, if you close the shell window, the Maqetta server process will stop running, at which point you can't run http://localhost:50000/maqetta anymore.
