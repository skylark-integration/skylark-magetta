Mobile theme editor basic tests

Mobile theme cloning, saving and opening

Steps

  1. You'll want to setup a virgin user environment by creating a whole new user login or resetting the existing user login (e.g., delete users/ folder, restart server). Clear the browser cache. Launch the Maqetta server. Login if necessary.
  2. Bring up the New Theme dialog multiple times to clone "custom" into "mymobiletheme" (or whatever), "android" into "android1" (or whatever), "iphone" into "iphone1" (or whatever), "ipad" into "ipad1" (or whatever) and "blackberry" into "blackberry1" (or whatever).
  3. Close two of the new theme editor windows by clicking on the "x" on the editor tab. Then reopen those theme editor tabs using the Open Theme... command.

Expected results

  1. A new theme editor tab should open for each of your new themes after issuing the New Theme command.
  2. The theme editor should show a global section at the top and a TabContainer at bottom where you can click on tabs to go to different groups of specific widgets.
  3. The cloned theme's look should be exactly the same as the original theme (i.e., the theme that was clone.)
  4. The Open Theme... command should show a list of all cloned themes and be able to open the theme editor on any of the cloned themes.

Global theming

Steps

  1. Do the following operations for each of your newly cloned themes:
    1. From the states tab select the Normal state. All widgets should be enabled
    2. Click to select the "Page backgrounds and Fonts" widget at the top of the editing window. In the Styling palette, make sure you can change background-color and background-image using a gradient. (You'll need to click on the "..." button to bring up the Background dialog in order to create/edit gradients.) Also in the Styling palette, change some combination of font-family, font-size, font-color, and boldness.
      You should see the effect on the canvas as you change properties. Some of the property changes such as font-family and font-size should affect multiple widgets.
    3. Click to select the "Heading Background" widget. In the Styling palette, change background color, background-image (using gradients in some cases), and some font properties.
      You should see the effect on the canvas as you change properties. Some of the property changes such as font-family and font-size should affect multiple widgets. Be sure to click on all of the tabs of the TabContainer surrounding the bottom-half of the mobile theme editor to see the effect of global styling changes across multiple widgets.
    4. Click to select the "Button Background" widget. In the Styling palette, change background color, background-image (using gradients in some cases), and some font properties.
      You should see the effect on the canvas as you change properties. Some of the property changes such as font-family and font-size should affect multiple widgets. Be sure to click on all of the tabs of the TabContainer surrounding the bottom-half of the mobile theme editor to see the effect of global styling changes across multiple widgets.
    5. Click on the Save icon on the main toolbar to save the new file.
    6. Click on the X close button for the editor tab for this file
    7. Re-open the theme editor for this theme from the Open menu.
    8. Issue a Create HTML File command, and set composition type to one of the mobile devices.
    9. Add some Dojo Mobile widgets (e.g., View, Heading, TabBar, Button, Switch) to the canvas.
    10. Click on the Switch Theme icon on the toolbar and switch to this new mobile theme.

Expected results

  1. As you change properties in the theme editor, the canvas should update so that the widgets reflect the styling changes you made.
  2. After save and reopen, the saved theme should open and its contents should be the same as how it was when you clicked on the Save button.
  3. After you have created your new HTML file with the widgets, you should see that the widgets have the styling changes you made in the theme editor.

Primitive widget theming and states

Steps

  1. From the states tab select the Normal state. All widgets should be enabled
  2. Click to on a few widgets and attempt to change Styling properties, particularly backgrounds, borders, fonts, and padding. For example, Heading, Button and TextBox.
    In most cases, you should see the effect on the canvas as you change properties. Note that there are various complexities with the theme editor, so not every CSS property can be changed for every widget. Usually, you are able to change backgrounds, borders and fonts.
    Another complexity involves backgrounds. If there is a background-image (including gradients) and it's fully opaque, it might entirely obscure the background-color..
  3. Change the state on the States palette from Normal to Selected. Change font and background on Button when in Selected state.
  4. Change properties on various widgets in various states.
  5. Click on the Save icon on the main toolbar to save the modified theme.
  6. Click on the X close button for the editor tab for this file
  7. Re-open the theme editor for this theme
  8. Create a new HTML file with composition type of one of the mobile devices (e.g., iphone). Before clicking Create, click on the "Theme..." button. In the Select theme dialog, for mobile theme, change "(device-specific)" to the name of your new theme. Then back in the New HTML file dialog, click Create.
  9. Add various widgets to the page for which you modified styling in the theme editor.

Expected results

  1. After you have created your new HTML file with the widgets, you should see that the widgets have the styling changes you made in the theme editor.

Subwidget theming

Steps

  1. If necessary, re-open the theme editor for one of your newly cloned mobile themes from the Open menu.
  2. Go to the "Rects and Lists" section at the bottom and click to select on the "Item 1" widget.
  3. With Widget Outer Container checked as the current subwidget, go to the Borders section of the Properties palette, choose "Show: props", and then set border-color to a different color.
    The border of the ListItem should reflect the new color.
  4. Click on the ListItem subwidget. Change background-color to pink.
    Both ListItems should show the new background color.
  5. Click on the MoveTo subwidget. Change border-color to green.
    The arrow should now be green.
  6. Click-select other widgets and find other widgets that have subwidgets. Try changing various properties.
    In some cases, you will see the effect on the canvas as you change properties, but in other cases not. Note that there are various complexities with the theme editor, so not every CSS property can be changed for every widget or sub-widget.
  7. Click on the Save icon on the main toolbar to save the modified theme.
  8. Click on the X close button for the editor tab for this file
  9. Re-open the theme editor for this theme
  10. Create a new HTML file with composition type of one of the mobile devices (e.g., iphone). Before clicking Create, click on the "Theme..." button. In the Select theme dialog, for mobile theme, change "(device-specific)" to the name of your new theme. Then back in the New HTML file dialog, click Create.

Expected results

  1. As you change properties in the theme editor, the canvas should update so that the widgets reflect the styling changes you made.
  2. After save and reopen, the saved theme should open and its contents should be the same as how it was when you clicked on the Save button.
  3. After you have created your new HTML file with the widgets, you should see that the widgets have the styling changes you made in the theme editor.
