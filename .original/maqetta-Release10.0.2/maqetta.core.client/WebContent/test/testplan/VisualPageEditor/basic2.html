Visual page editor basic tests (part 2)

Widgets, images, open/save (relative layout)

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to setup a virgin user environment by creating a whole new user login.
  2. Click on the "New file" icon on the main toolbar. Choose any filename.
  3. Drag a few primitive widgets onto the canvas (e.g., Button, TextBox, Calendar).
  4. Drag an image onto the canvas (e.g., SampleBanner.jpg).
  5. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  6. Click on the "X" close button for the editor tab for this file
  7. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. The saved file should open and its contents should be the same as how it was when you clicked on the Save button.
  2. The canvas should show the widgets and image(s) that you dragged onto the canvas.
  3. The Outline palette should show the widgets and image(s).

Container widgets (continuation of previous)

Steps

  1. Re-open the file created in the previous steps.
  2. Drag a few container widgets onto the canvas (e.g., ContentPane, Accordion, TabContainer).
  3. Drag an HTML DIV widget onto the canvas.
  4. Drag a couple of primitive widgets into each container (e.g., Button, TextBox, HTML LABEL, HTML SPAN).
  5. Drag a couple of container widgets inside other containers to create nested containers (e.g., Accordion inside of a TabContainer). Drag primitive widgets into the nested container widget.
  6. Click on the "Save" icon on the main toolbar to save the new file. The asterisks that indicates that the file is "dirty" should go away.
  7. Click on the "X" close button for the editor tab for this file
  8. In the Files palette, double-click on the file you just created to re-open the file.

Expected results

  1. The saved file should open and its contents should be the same as how it was when you clicked on the Save button.
  2. The canvas should show the widgets and image(s) that you dragged onto the canvas.
  3. The Outline palette should show the widgets and image(s). Within the Outline palette, the widgets that you dragged into containers should appear as nested children of their container widget.

Selecting, moving, resizing, and deleting widgets

Steps

  1. Re-open the file created in the previous steps.
  2. Click to select various widgets.
  3. Move selected widgets to different locations. With relative layout, you should be able to re-order widgets within the flow via drag/drop. With absolute layout, you should be able to change widget position.
  4. Some widgets are resizable and show resize knobs when selected. For example, ContentPane and Calendar. Drag the knobs to resize some widgets.
  5. Delete a widget by selecting it and then invoking the "Delete" command from the context menu.

Expected results

  1. You should be able to select, move, resize and delete widgets.

Design/source views (continuation of previous)

Steps

  1. Re-open the file created in the previous steps.
  2. Click on the Split View icon (top/bottom views).
  3. In the source editor, select and delete one of the widgets.
  4. In the source editor, select, copy, paste one of the widgets to clone a second version of that widget.
  5. Change some textual content (e.g., a button label) and change the width property of a widget.

Expected results

  1. The visual image of the document should reflect immediate changes with every change to the source code of the document. Changes should occur with each keystroke.

Absolute layout

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Turn on absolute layout by clicking on the Switch Layout icon.
  3. Repeat all of the tests that you ran above in relative layout mode.

Expected results

  1. Similar results as for relative layout, except (of course) widgets will be positioned at absolute locations instead of showing a flow layout.

Cut/copy/paste/delete from context menu

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Add a Button, an Accordion, ComboBox, Calendar, SPAN, DIV and a few LABEL elements to the page.
  3. Verify that the context menu commands cut, copy, paste and delete are working on individually selected widgets.
  4. Verify that the context menu commands cut, copy, paste and delete are working with multiple selection.

Expected results

  1. (See notes above for expected behavior)

Surround with (<div> and <span>)

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. Add a Button, an Accordion, ComboBox, Calendar, SPAN, DIV and a few LABEL elements to the page.
  3. Individually select a couple of widgets (both Dojo and HTML) and issue a "Surround with <div>" command from the context menu.
  4. Individually select a couple of widgets (both Dojo and HTML) and issue a "Surround with <span>" command from the context menu.
  5. Multiply select widgets that share the same parent (including both Dojo and HTML) and issue a "Surround with <div>" command from the context menu.
  6. Multiply select widgets that share the same parent (including both Dojo and HTML) and issue a "Surround with <span>" command from the context menu.

Expected results

  1. In both the Outline palette and in source view, the selected widgets should have become the children of DIVs or SPANs, as appropriate.

Reorder commands (Move to front, Move to back, Move forward, Move backward)

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. New HTML File - hi-fi page
  3. Bring up context menu in both canvas and Outline palette
    • You should menu commands Move to front, Move to back, Move forward, Move backward
    • All four commands should be disabled
  4. Add a Button
  5. Bring up context menu in both canvas and Outline palette
    • You should menu commands Move to front, Move to back, Move forward, Move backward
    • All four commands should be disabled
  6. Switch to absolute positioning
  7. Add a CheckBox, Calendar, ColorPalette, Rectangle, and a Image widget from clipart section
  8. Add a DIV. Inside of the DIV, drop an HTML Button and an HTML Select
  9. Open up Outline palette (we will be looking at it repeatedly)
  10. Select the CheckBox
  11. Bring up context menu in both canvas and Outline palette
    • You should menu commands Move to front, Move to back, Move forward, Move backward
    • Because the CheckBox is the first absolute child, only Move to front and move forward should be enabled.
  12. Issue a move forward command
    • In the Outline palette, the Checkbox should now have moved after the Calendar
  13. Issue an Undo command
  14. Select the DIV
  15. Bring up context menu in both canvas and Outline palette
    • You should menu commands Move to front, Move to back, Move forward, Move backward
    • Because the DIV is the last absolute child, only Move to back and move backward should be enabled.
  16. Issue a move backward command
    • In the Outline palette, the DIV should now have moved before the clipart Image widget
  17. Issue an Undo command
  18. Multi-select the Calendar and Rectangle.
  19. Bring up context menu in both canvas and Outline palette
    • You should menu commands Move to front, Move to back, Move forward, Move backward
    • Because the Calendar and Rectangle are not adjacent children, only Move to front and move to back should be enabled.
  20. Issue a move to back command
    • In the Outline palette, the Calendar and Rectangle should be the first two children of the BODY
  21. Issue an Undo command
  22. Multi-select the Calendar and ColorPalette.
  23. Bring up context menu in both canvas and Outline palette
    • You should menu commands Move to front, Move to back, Move forward, Move backward
    • Because the Calendar and Rectangle are now adjacent children, all four reorder commands should be enabled.
  24. Issue a move forward command
    • In the Outline palette, the Calendar and ColorPalette should be immediately after the Rectangle

Expected results

  1. As described above.

Select/unselect commands

Steps

  1. Standard setup things: Clear the browser cache. Launch the Maqetta server. Login if necessary. In some cases, you'll want to a virgin user environment by creating a whole new user login.
  2. New HTML File - hi-fi page
  3. Bring up context menu in both canvas and Outline palette
    • You should menu commands Select parent, Select ancestor..., and Unselect all
    • All three commands should be disabled
  4. Add a TabContainer. Add an Accordion inside of TabContainer. Add a Button and Checkbox inside of Accordion
  5. Select the Button and bring up context menu on canvas
    • All three commands should be enabled
  6. Issue a Select parent command
    • The ContentPane for Accordion should be selected
  7. Consecutively issue 3 more Select parent commands, and observe what gets selected
    • The Accordion should be selected, then the ContentPane for TabContainer, than TabContainer
  8. Select the Button and bring up context menu on canvas and issue a Select ancestor command
    • The Select ancestor dialog should appear
    • The list of options in the dropdown should show ContentPane, Accordion, ContentPane, TabContainer
  9. From the menu, choose the Accordion
    • The Accordion should now be selected
  10. Select the Button and Checkbox, bring up context menu on canvas and issue an Unselect all command
    • Nothing should be selected

Expected results

  1. As described above.
